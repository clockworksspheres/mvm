# Virtual Machine Manage (vmm)

## What is Virtual Machine Manage

An OO factory that pumps out concrete classes
to manage virtual machines based on the hypervisor
being used.

The hypervisor needs to be running prior to use
of this code.

Currently this is designed for macOS and Windows, with the 
following hypervisors:

 * VmWare Fusion and Workstation
 * Virtualbox
 * UTM
 * HyperV

Linux support on the roadmap for:

 * VmWare Fusion and Workstation
 * Virtualbox
 * KVM
 * Xen
 * QEMU

## Types of hypervisors

(generated by search.brave.com)

[Type 1 and Type 2 Hypervisors](https://www.geeksforgeeks.org/system-design/hypervisor/)

### **Type 1 hypervisor** (also known as a **bare metal hypervisor**) runs directly on the physical hardware of a server, without requiring a host operating system. It has direct access to the underlying hardware, enabling high performance, better security, and efficient resource management. These are commonly used in enterprise data centers and cloud environments for mission-critical workloads.

### **Type 2 hypervisor** (also known as a **hosted hypervisor**) operates as an application installed on top of a conventional operating system (like Windows or Linux). It relies on the host OS to manage hardware resources, which introduces a performance overhead. Type 2 hypervisors are easier to set up and are ideal for development, testing, and personal use.

### Key differences:
- **Performance**: Type 1 offers superior performance due to direct hardware access.
- **Security**: Type 1 has a smaller attack surface and is more secure.
- **Use Case**: Type 1 is used in large-scale, production environments; Type 2 is suited for desktops and development.
- **Examples**:  
  - Type 1: **VMware ESXi**, **Microsoft Hyper-V**, **KVM**
  - Type 2: **Oracle VM VirtualBox**, **VMware Workstation**, **VMware Fusion**, **Parallels Desktop**, **UTM**

## Vmware VM Notes

Type 2 hypervisor

### VMware Fusion

The default path "/full/path/to/my.vmx" for managing 
VMware Fusion VM's, should look like:

"/Users/<username>/Virtual Machines.localized/MyVm.vmwarevm/MyVm.vmx"

Unless the user has saved the VM to a different location.
